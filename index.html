原型的理解

在JS中，__proto__可以理解为“爸爸”，prototype则是祖辈的传统手艺。爸爸每生出一个孩子，孩子就会具备从爸爸那沿袭来的手艺（公用方法）。

在上图中，总共有三条主线。

  • 第一条是蓝色箭头部分。var n = new Number()的__proto__指向Number.prototype，而Number.prototype中，也有一个__proto__ ，指向了Object.prototype。这一条线被称为原型链，因为它可以通过n.__proto__.__proto__的形式，一直追溯到祖先（Object），结构上像一根链条。
  • 第二条是橙色部分。Number.__proto__和Object 这常常使人混淆，因为Number.prototype中，也有一个__proto__，但是它们两个的意思不同。Number.__proto__ 存在，是因为Number本身是一个函数，它应该指向Funtion.prototype。（参见图中与4，5节点相关的箭头）
  • 第三条是黑色箭头部分，是一种特殊的情况，详见下文。

Function对象

Function对象是一个极其例外的对象，其他对象都有一个可追溯的原型对象存在，但是Function的原型对象就是它自己，因为Function本身也是一个函数。所以就存在这种特殊情况。

JS中的数据类型

​ 在JS中，总共有7种数据类型。

  • number

  • boolean

  • string

  • symbol

  • null

  • undefined

  • object

​ 这7种数据类型，可以大致分为三个类别，第一类是number, boolean, string, symbol等基本类型; 第二类是null, undefined; 第三类就是object. 把object单独分为一类是因为它包含的范围太大了。

基本类型

number

​ JS与JAVA相比，数据类型大大减少了。比如数字，JAVA中整数根据位数不同，分为byte, short, int, long等，还有浮点数float和double，而在JS中被笼统地归为number了。其他类型也有类似的情况。JS中所有数字, 都以64位的浮点数进行存储的，包括整数。也因为如此，在进行小数加减的时候，要注意结果小数点后面的位数。

​ number使用前缀区分数字的进制，前缀“0”代表八进制，“0b”代表二进制， “0x”代表16进制。涉及到与用户交互的层面时，都会被转化为十进制。

null & undefined

​ 这两个同时存在是一个错误，意思完全相同。但是既然存在了就只能接受它。它俩的使用有一个约定俗成的规矩，就是空对象用null，未定义基本类型使用undefined。undefined比null更为常见。

这也意味着，字符串的length属性是不可靠的，不一定等于字符的数量。

CSS基本布局技巧

左右结构

​ 使用悬浮属性可以达到左右结构的效果。具体的实现方法是给所有标签加上float: left 或者 float: right。但是浮动后会有副作用，因为浮动元素脱离了文档流，盒子的宽度会缩减到最小，并且会和下方未浮动的元素重合。重合的问题可以通过clear属性解决。具体的方法有三种：

  1. 使用一个空的div，给这个div施加clear: both属性。但是这个方法，会给页面增加无用的标签，容易让人费解其作用。
  2. 给父元素加overflow: hidden属性。这个属性本身是文档流内容超出父元素边框时的处理方式，但它有个自动清理子元素的副作用，所以可以用来解决问题。
  3. 给父元素加clearfix类。这个方法是通过给div的::after伪元素施加clear属性，来达到清理的目的。这个方法与第一个方法的原理类似，但利用的是伪元素，不会给页面添加没有意义的div标签，是最为推荐的做法。

左中右结构

​ 左中右结构属于左右结构的一种特殊形式，所以基本做法与左右结构相同。额外的操作是给三个元素设置width属性，并使其和为100%，达到刚好占用一行的目的。最好给三个元素再加上box-sizing: border-box 属性，避免给元素加了margin或者padding属性后，元素宽度溢出而导致自动换行。

水平居中

​ 如果是内联元素，实现居中只要在父元素上加text-align: center 就可以了。行级元素则稍微复杂一些，实现水平居中主要有两种方法。

  • margin: 0 auto

    使用这个属性虽然可以使元素居中，但是需要指定元素的宽度。如果不指定元素宽度，则会默认占据100%的行宽，这样margin属性也就失去意义了。

  • 第二种方法是将display修改为inline-block。这样元素就同时具备的inline和block的一部分特性，可以通过text-align: center来实现居中了。

第二种方法是更为推荐的方法，因为给元素设置定死的width和height属性是许多bug的来源，应该尽量避免去设置这两个属性。使用JS来根据文档流大小，动态修改宽高也是一种可行的方案。

垂直居中

​ 在inline元素中，由于高度无法通过height指定，只能使用line-height来改变，所以文字总是居中的。
​ 在块级元素中，未指定height属性时，可以使用上下相等的padding实现居中。这样实现的好处是，任意改变元素的内容，总是垂直居中的。

HTML标签基础

a

a标签全称 anchor，意思是锚，是最常用的标签之一。它常用于跳转链接，也可以用于定位当前页面的某个元素。

href

href 的全称是 Hypertext Reference，是 a 标签最为重要的属性，它代表点击该标签后跳转的目标。内容可以是相对路径，也可以是绝对路径。href 的取值可以是以下内容。

  • page.html

    引用了同一目录下，名为 anotherPage.html 的网页，点击后将跳转到该网页，使用了相对路径，浏览器会自动补全为当前网页所在目录。

  • http://www.website.com/page.html

    这个属于绝对引用，于相对路径而言更易理解，但是灵活性较差。如果我们移动了项目文件夹，这个引用将失效，浏览器会回应404状态码。而相对路径没有这个问题。

  • //page.html

    这种方式并未规定访问的协议，浏览器会默认采用当前页面的协议进行不全。例如，使用file://协议访问时，点击该链接后，会继续使用file://协议进行访问。Http协议同理。

  • #xxx

    当xxx的内容是某个元素的id时，滚动条会转至该元素所在位置，用于页面位置的跳转。如果xxx为空，则跳转至页首。

  • javascript: ;

    该写法被称为伪协议。因为 javascript 这个单词所在位置，原本应是file or http，但是javascript是脚本而不是协议名称，所以被称为伪协议。冒号后面如果加上JS代码，就会在点击a标签时执行。直接加；而不是JS代码的话，就不会有任何效果产生。这种写法常用于解决一些奇怪的需求，比如点击链接后，但却不作出任何反应，或者是以下奇怪的代码：

  3. _top 在顶层窗口中打开
  4. _parent 在父窗口中打开

form

​ form 表单是十分重要的数据提交方式，也是网页中以post方式提交数据的唯一途径。在form中，使用method属性定义表单提交的方式。method 仅支持 post 和 get 方式，如果填写其他方式，如delete，会被自动转化成 get 方式请求数据。post方式是最为常用的属性。

​ 以post方式提交的数据，会经过 urlEncoded 这个步骤，英文字母不会发生改变，中文会被转化为 utf-8 编码。比如“密码”两个字，会被编码成E5 AF 86 和 E7 A0 81，然后每个字节被%包围，最终效果是%E5%AF%86%E7%A0%81 。中文冒号“：”会被编码成 %EF%BC%9A 。

​ action属性定义了form提交的目标地址，和a标签的href属性一样，可以使用相对路径和绝对路径。

  • text
  • radio
  • checkbox
  • file
  • password
  • submit

另外，textarea和select虽然与上述Input元素形式不太相同，但也是在表单中常用的标签。

Git入门

git init

git init 用于创建本地git仓库，在需要创建的目录下使用后，会生成一个名为 .git 的文件夹用于保存数据。

git add

在本地仓库创建之后，对任何文件进行修改，都会使其状态发生改变。

如图中所示，使用git status -sb 命令查看状态，红色的M表示未处理的，内容被修改过的文件；红色的？？表示新创建的文件。

然后使用 git add 文件目录 的形式，就可以将未处理的文件添加到待提交列表中。

此时两个修改的文件状态变为绿色，下一步就可以使用git commit进行提交了。

git commit -v

git commit 命令即将add后的文件从缓存区提交到本地仓库。-v是指verbose, 繁琐的，给出的提示信息会更详细。

另外 -m 也是一个常用的参数，通常使用 git commit -v -m “一些提交相关的信息” 的形式来进行提交。
