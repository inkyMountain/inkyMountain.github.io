JS中的数据类型

​ 在JS中，总共有7种数据类型。

  • number

  • boolean

  • string

  • symbol

  • null

  • undefined

  • object

​ 这7种数据类型，可以大致分为三个类别，第一类是number, boolean, string, symbol等基本类型; 第二类是null, undefined; 第三类就是object. 把object单独分为一类是因为它包含的范围太大了。

基本类型

number

​ JS与JAVA相比，数据类型大大减少了。比如数字，JAVA中整数根据位数不同，分为byte, short, int, long等，还有浮点数float和double，而在JS中被笼统地归为number了。其他类型也有类似的情况。JS中所有数字, 都以64位的浮点数进行存储的，包括整数。也因为如此，在进行小数加减的时候，要注意结果小数点后面的位数。

​ number使用前缀区分数字的进制，前缀“0”代表八进制，“0b”代表二进制， “0x”代表16进制。涉及到与用户交互的层面时，都会被转化为十进制。

null & undefined

​ 这两个同时存在是一个错误，意思完全相同。但是既然存在了就只能接受它。它俩的使用有一个约定俗成的规矩，就是空对象用null，未定义基本类型使用undefined。undefined比null更为常见。

string

​ JS由于历史原因，字符串仅支持两个字节的字符，如果字符字节大于2，会被认为是两个字符。比如“𝌆”。

这也意味着，字符串的length属性是不可靠的，不一定等于字符的数量。

object

​ 对象是一些有关联数据的集合体，就像整理房间的收纳盒。 它的表现形式是键值对，即key: value的组合。
