HTML标签基础

a

a标签全称 anchor，意思是锚，是最常用的标签之一。它常用于跳转链接，也可以用于定位当前页面的某个元素。

href

href 的全称是 Hypertext Reference，是 a 标签最为重要的属性，它代表点击该标签后跳转的目标。内容可以是相对路径，也可以是绝对路径。href 的取值可以是以下内容。

  • page.html

    引用了同一目录下，名为 anotherPage.html 的网页，点击后将跳转到该网页，使用了相对路径，浏览器会自动补全为当前网页所在目录。

  • http://www.website.com/page.html

    这个属于绝对引用，于相对路径而言更易理解，但是灵活性较差。如果我们移动了项目文件夹，这个引用将失效，浏览器会回应404状态码。而相对路径没有这个问题。

  • //page.html

    这种方式并未规定访问的协议，浏览器会默认采用当前页面的协议进行不全。例如，使用file://协议访问时，点击该链接后，会继续使用file://协议进行访问。Http协议同理。

  • #xxx

    当xxx的内容是某个元素的id时，滚动条会转至该元素所在位置，用于页面位置的跳转。如果xxx为空，则跳转至页首。

  • javascript: ;

    该写法被称为伪协议。因为 javascript 这个单词所在位置，原本应是file or http，但是javascript是脚本而不是协议名称，所以被称为伪协议。冒号后面如果加上JS代码，就会在点击a标签时执行。直接加；而不是JS代码的话，就不会有任何效果产生。这种写法常用于解决一些奇怪的需求，比如点击链接后，但却不作出任何反应，或者是以下奇怪的代码：

target

target 属性规定了新链接打开的位置，它的值包括：

  1. _blank 以新的标签页打开页面
  2. _self 在当前标签页打开页面
  3. _top 在顶层窗口中打开
  4. _parent 在父窗口中打开
  5. 某个iframe标签的name属性

form

​ form 表单是十分重要的数据提交方式，也是网页中以post方式提交数据的唯一途径。在form中，使用method属性定义表单提交的方式。method 仅支持 post 和 get 方式，如果填写其他方式，如delete，会被自动转化成 get 方式请求数据。post方式是最为常用的属性。

​ 以post方式提交的数据，会经过 urlEncoded 这个步骤，英文字母不会发生改变，中文会被转化为 utf-8 编码。比如“密码”两个字，会被编码成E5 AF 86 和 E7 A0 81，然后每个字节被%包围，最终效果是%E5%AF%86%E7%A0%81 。中文冒号“：”会被编码成 %EF%BC%9A 。

​ action属性定义了form提交的目标地址，和a标签的href属性一样，可以使用相对路径和绝对路径。

input

type

较为常用的input元素类型有：

  • text
  • radio
  • checkbox
  • file
  • password
  • submit

另外，textarea和select虽然与上述Input元素形式不太相同，但也是在表单中常用的标签。
