Git入门

git init

git init 用于创建本地git仓库，在需要创建的目录下使用后，会生成一个名为 .git 的文件夹用于保存数据。

git add

在本地仓库创建之后，对任何文件进行修改，都会使其状态发生改变。

如图中所示，使用git status -sb 命令查看状态，红色的M表示未处理的，内容被修改过的文件；红色的？？表示新创建的文件。

然后使用 git add 文件目录 的形式，就可以将未处理的文件添加到待提交列表中。

此时两个修改的文件状态变为绿色，下一步就可以使用git commit进行提交了。

git commit -v

git commit 命令即将add后的文件从缓存区提交到本地仓库。-v是指verbose, 繁琐的，给出的提示信息会更详细。

另外 -m 也是一个常用的参数，通常使用 git commit -v -m “一些提交相关的信息” 的形式来进行提交。
